凡是做过两次以上的操作，必须转为一键脚本。AI 不推理步骤。

**Why:** 每次让 Worker 推理"怎么部署"浪费大量 token（理解指令→推理步骤→逐条执行→分析输出），而且容易出错（比如选错目录、用错方式）。脚本是验证过的最佳实践，AI 推理是不可靠的记忆。

**How to apply:**
1. **第一次**：AI 按 Skill 或指令执行，探索正确流程
2. **验证 OK 后**：立即把流程写成 `scripts/xxx.sh` 一键脚本
3. **以后**：Worker 只执行 `bash scripts/xxx.sh` → 读输出 → 判断成功/失败 → 回报
4. AI 的角色变成**路由器**（识别意图→调脚本）+ **确认器**（读输出→汇报），不是执行器
5. 类似 Jenkins — 脚本做事，人（AI）确认

**适用范围：**
- 部署（deploy-erp-admin.sh ✅ 已脚本化）
- 服务器健康检查
- SSL 证书申请
- DNS 配置
- 任何 SSH 操作序列
